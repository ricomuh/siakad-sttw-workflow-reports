## 🔍 Temuan & Issues

### ⚠️ Issue: Auth State Expired (E2E)

**Deskripsi**: Seluruh 13 screenshot E2E menampilkan halaman login (`/login`), bukan halaman target. Semua file berukuran identik ~360 KB — konsisten dengan redirect massal.

**Root Cause**: File auth state di `tests/e2e/.auth/` (mahasiswa.json, kaprodi.json, waket1.json) sudah expired. Playwright storage state tidak bisa melewati middleware auth Laravel karena session/token sudah kadaluarsa.

**Dampak**: E2E tests tidak bisa memvalidasi UI sebenarnya — semua screenshot useless. Test assertion (`expect(hasCourse || hasEmpty).toBeGreaterThan(0)`) tetap bisa PASS karena login page memiliki konten DOM, tapi bukan konten yang diinginkan.

**Rekomendasi**:
1. Regenerate auth state dengan menjalankan ulang `tests/e2e/setup/auth.setup.ts`
2. Atau gunakan `globalSetup` Playwright untuk login otomatis via API sebelum test suite
3. Tambahkan assertion awal di setiap test: `await expect(page).not.toHaveURL(/\/login/)` untuk catch auth failure early

### ℹ️ Temuan: QA Planner Count Discrepancy

QA Planner mencatat 9 tests untuk UjianMahasiswaTest + 9 untuk PenjamuMonitoringDetailTest, aktualnya 10 + 8. Total tetap 24.

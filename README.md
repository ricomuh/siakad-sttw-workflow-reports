## How to read a report

Every `REPORT.md` follows the workflow-reporter format:

1. **Header** — Tanggal, Role, Modul, Fitur, Status (✅/⚠️/❌)
2. **Deskripsi Workflow** — what the user is trying to accomplish
3. **Ringkasan** — high-level findings
4. **Langkah-langkah** — step-by-step screenshots from sidebar navigation through each interaction
5. **Skenario Alternatif** — variant flows (different form options, period states, role views)
6. **Temuan & Masalah** — bugs/UX issues observed during recording (categorised: `styling`, `server-error`, `permission`, `no-data`, `broken-ui`, `missing-feature`, `missing-sidebar`, etc.)
7. **Catatan** — observations, data preparation, known limitations

Screenshots are stored in each report's `screenshots/` subdirectory.

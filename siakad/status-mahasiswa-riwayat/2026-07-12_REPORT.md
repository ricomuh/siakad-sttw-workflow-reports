## Ringkasan

Fitur pelacakan riwayat perubahan status mahasiswa. Setiap kali status mahasiswa berubah (Aktif → Cuti, Aktif → Lulus, dll), sistem otomatis mencatat perubahan ke tabel `status_mahasiswa_riwayat` via Observer. Admin dapat melihat timeline riwayat status dari halaman detail mahasiswa.

## Langkah-langkah

### 1. Admin — Daftar Mahasiswa

Admin navigasi ke Data Mahasiswa → melihat daftar mahasiswa dengan status terkini.

### 2. Admin — Detail Mahasiswa

Klik mahasiswa → halaman detail menampilkan biodata, KRS, KHS, dan tombol "Riwayat Status" (ungu) di header.

### 3. Admin — Timeline Riwayat Status

Klik "Riwayat Status" → halaman timeline menampilkan semua perubahan status dalam urutan kronologis (terbaru di atas). Setiap entri menampilkan:
- Status lama → Status baru (dengan badge warna)
- Tanggal & waktu perubahan
- User yang melakukan perubahan
- Pagination jika riwayat > 10

### 4. Observer — Auto-track Perubahan

Saat admin mengubah status mahasiswa (edit → simpan), `MahasiswaObserver::updated()` otomatis mendeteksi perubahan field `status_mahasiswa` dan membuat record baru di `status_mahasiswa_riwayat`. Tidak ada perubahan status yang terlewat.

## Catatan

- `users.id` menggunakan `char(36)` (UUID) sehingga `created_by` menggunakan tipe yang sama dengan manual FK constraint
- Observer hanya trigger saat field `status_mahasiswa` berubah — perubahan field lain tidak membuat record baru
- Timeline menggunakan warna badge: Aktif=hijau, Cuti=kuning, Lulus=biru, Drop Out=merah, dll

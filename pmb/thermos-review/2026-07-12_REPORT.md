## Langkah-langkah

### 1. Issue #3 — resolveStep() Side-effect Fix

`resolveStep()` pada service PMB sebelumnya tidak jelas apakah method ini pure (hanya baca/hitung) atau impure (tulis ke DB). Ini adalah side-effect tersembunyi yang menyebabkan kebingungan dan potensi bug saat method dipanggil di konteks yang tidak mengharapkan write.

Fix yang diterapkan:
- Tambah PHPDoc block yang menjelaskan behaviour masing-masing method
- Rename: `resolveStepPersist()` untuk variant yang menulis ke DB (side-effect eksplisit), `calculateStep()` untuk variant pure yang hanya menghitung tanpa side-effect

Dengan penamaan eksplisit ini, caller langsung tahu apakah memanggil method yang akan menulis ke DB atau tidak.

### 2. Issue #4 — Hapus Hardcoded DB Credentials

`tests/CreatesApplication.php` memiliki fallback:

```php
// BEFORE (berbahaya)
DB::connection()->getPdo()->exec('...');
// dengan fallback: username='root', password='password'
```

Fallback ini berbahaya karena:
1. Jika `.env.testing` tidak diset, test akan diam-diam connect ke DB dengan credentials default — berpotensi korupsi data di environment yang menggunakan credentials tersebut
2. Mengekspos pattern "default insecure credentials" di codebase

Fix: hapus fallback — credentials wajib eksplisit dari `.env.testing`. Jika `.env.testing` tidak lengkap, test akan fail dengan error yang jelas daripada diam-diam menggunakan credentials tidak aman.

Test `CreatesApplicationEnvTest` memverifikasi bahwa config DB test sekarang dibaca dari environment variable, bukan dari hardcoded fallback.
